# Claude Code 토큰 사용량 90% 절감하는 3가지 실전 기법

중간 규모 프로젝트에서 Claude Code 토큰의 80~90%가 코드 탐색에 낭비되는 문제를 해결하는 3가지 기법.

---

## 기법 1: 기본 설정 최적화 (30~40% 절감)

- **`.claudeignore` 파일**: 불필요한 파일(node_modules, .next, dist 등)을 읽지 않게 차단
- **`claude.md` 200줄 이내 유지**: 매 대화마다 전체 로드되므로 핵심만 남기고 상세 규칙은 별도 파일로 분리
- **세션 관리**: `/compact`로 긴 대화 요약, 새 작업은 `/clear`로 초기화
- **MCP 서버 최소화**: 불필요한 서버 차단 (연결된 서버마다 도구 정의 토큰 발생)

## 기법 2: 코드베이스 사전 인덱싱 — qmd (탐색 토큰 92% 절감)

문제: Claude Code가 파일을 찾을 때 glob → grep → 파일 읽기 순으로 반복 탐색하며 토큰 낭비.

해결: [[entities/qmd]] 도구로 프로젝트를 미리 인덱싱하고 MCP 서버로 연결.

실제 절감 수치:
| 작업 | 기존 | qmd 적용 후 | 절감 |
|---|---|---|---|
| 인증 로직 탐색 | 2,700 토큰 | 250 토큰 | 91% |
| 결제 버그 수정 | 3,950 토큰 | 400 토큰 | 90% |
| DB 스키마 파악 | 6,000 토큰 | 400 토큰 | 93% |

`claude.md`에 "파일 읽기 전 항상 qmd로 먼저 검색" 규칙 한 줄 추가로 자동화.

## 기법 3: 3-에이전트 팀 구조

단일 에이전트가 설계·구현·검토를 모두 담당하면 범위를 벗어난 탐색과 구현이 발생. 이를 역할 분리로 해결.

| 역할 | 책임 |
|---|---|
| **아키텍트** | 전체 시스템 이해, 작업 계획·브리프 작성, 수정할 파일 지정 |
| **빌더** | 브리프에 명시된 내용만 구현. 브리프 외 작업 금지 |
| **리뷰어** | 완성 코드 검증, 문제 시 구체적 피드백과 함께 반려 |

오픈소스 템플릿: [three-man-team](https://github.com/russelleNVy/three-man-team)

## 추가: CLAUDE.md 토큰 절약 5개 규칙

1. 이미 읽은 파일은 다시 확인하지 않기
2. 불필요한 도구 호출 차단하기
3. 가능한 호출은 동시에 실행하기
4. 20줄 이상의 불필요한 출력은 서브 에이전트로 보내기
5. 사용자가 이미 설명한 내용을 다시 말하지 않기
